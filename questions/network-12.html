QUESTION 111#original_knowledge_0053

HTTPSで保護されたWebサイトについて、最も適切な説明はどれか。

  1. 0

    通信経路を保護するが、サイト内容の真実性は保証しない。

  2. 1

    httpsで始まるURLなら詐欺サイトではあり得ない。

  3. 2

    HTTPは暗号化され、HTTPSは暗号化されない。

  4. 3

    著作権侵害を自動的に防ぐ仕組みである。

正答と解説を確認

正答0. 通信経路を保護するが、サイト内容の真実性は保証しない。

解説

HTTPSは通信の保護を主目的とし、詐欺サイトでないことや商品品質まで保証するものではない。 HTTP(Hyper Text Transfer Protocol)はウェブブラウザとウェブサーバの間でのやり取りの手順、書式を決めるプロトコルである。 通信は平文（暗号化されていない文）で行われるため、悪意を持ったユーザに盗聴される可能性がある。 HTTPS(HTTP Secure)はSSL(Secure Sockets Layer)やその後継であるTLS(Transport Layer Security)プロトコルによって提供される暗号化された通信を行う。 URLがhttpsで始まるウェブサイトはSSL(Secure Sockets Layer)やその後継であるTLS(Transport Layer Security)を用いて暗号化されたセキュアな通信をウェブサーバとブラウザとの間で行っていることを意味する。 これにより、第三者に通信の傍受や改ざんがされにくくなる。 しかし、httpsであることはそのウェブサイトの信頼性や正確性を保証するものではない。

QUESTION 112#original_knowledge_0054

Webブラウザのキャッシュの主な目的として最も適切なものはどれか。

  1. 0

    毎回必ずデータを削除して再取得すること。

  2. 1

    以前取得したデータを再利用し、通信量や表示時間を減らすこと。

  3. 2

    閲覧履歴を全て外部に公開すること。

  4. 3

    WebサーバのIPアドレスを変更すること。

正答と解説を確認

正答1. 以前取得したデータを再利用し、通信量や表示時間を減らすこと。

解説

キャッシュは画像などを一時的に保存して再利用し、閲覧を高速化する。 ウェブブラウザがインターネット上のウェブページを表示する際、HTML、CSS、JavaScript、画像など多くのファイルをダウンロードする必要がある。 ウェブブラウザは受け取ったこれらのファイルをキャッシュ（貯蔵庫）に保存し、その後、画面に表示する。 再度、同じページを閲覧するときには、キャッシュからこれらのファイルを読み込む。 このようにすることで、ファイルを何度もダウンロードする必要がなくなり、ウェブページの読み込み時間を大幅に短縮することができる。 ただし、キャッシュはあくまで一時的なものであり、ウェブページの最新版を反映していない場合があるため注意が必要である。

QUESTION 113#original_knowledge_0055

HTTPステータスコード404が示すものとして最も適切なものはどれか。

  1. 0

    利用者のPCの電源が切れていること。

  2. 1

    要求したサーバが必ず停止していること。

  3. 2

    要求したWeb上の資源がサーバで見つからないこと。

  4. 3

    通信内容が暗号化されていること。

正答と解説を確認

正答2. 要求したWeb上の資源がサーバで見つからないこと。

解説

404は、指定したページやファイルが見つからないことを表す応答である。 404 Not Foundは、サーバ自体は存在するが、クライアント（ウェブブラウザ）が要求したリソース（ウェブページやコンテンツ）がサーバ上の指定された場所に存在しないことを示すHTTPステータスコード（エラーメッセージ）である。 このエラーメッセージは、リソースが削除・移動されていたとき、URLのパスの部分を打ち間違えてた場合などに発生する。 例：example.com/indexとすべきところをexample.com/imdexとしている。 ドメイン名が間違っているなどの理由で、指定されたサーバ自体が存在しない場合、ブラウザやOSによって異なるが「このサイトにアクセスできません」や「このページに到達できません」などのエラーメッセージが表示される。 つまり、404 Not Foundというエラーは「サーバが存在しない」ことを示すのでなく「要求されたリソースがサーバ上に見当たらない」ことを示すものである。

QUESTION 114#original_knowledge_0056

電子メールの配送について、最も適切な説明はどれか。

  1. 0

    送信者と受信者の端末が常に直接接続して配送する。

  2. 1

    受信者の端末が停止中なら直ちに永久消失する。

  3. 2

    メールアドレスにはドメイン部分が不要である。

  4. 3

    送信メールはメールサーバを経由し、宛先側のサーバに配送・保管される。

正答と解説を確認

正答3. 送信メールはメールサーバを経由し、宛先側のサーバに配送・保管される。

解説

受信者の端末が停止中でも、通常は宛先のメールサーバに保管され、後で取得できる。 電子メール（以下メール）は直接ユーザの端末に送られるわけではない。 ユーザの端末に直接メールを送る仕組みとした場合、その端末の電源が入っていなければ、メールを届けることができない。 そこで、メールをやり取りする専門のコンピュータ（メールサーバ）を用意し、常時稼働させておくことにする。 電子メールは必ずしも暗号化された通信で送られるわけではない。 メールの送受信に用いられるプロトコル（SMTP,POP,IMAPなど）には元来、暗号化機能がないためである。 そのため、これらのプロトコルを用いてメールを送受信した場合、その内容は第三者によって盗聴や改ざんの可能性がある。

QUESTION 115#original_knowledge_0057

プロキシサーバの主な役割として最も適切なものはどれか。

  1. 0

    利用者に代わって外部のWebサーバなどへのアクセスを中継すること。

  2. 1

    通信規則に基づいてパケットを許可・遮断すること。

  3. 2

    CPU内部の計算誤差を訂正すること。

  4. 3

    データベースの主キーを自動設定すること。

正答と解説を確認

正答0. 利用者に代わって外部のWebサーバなどへのアクセスを中継すること。

解説

プロキシサーバは内部の利用者と外部のサービスとの間で通信を代理・中継する。 プロキシサーバはクライアント（ウェブブラウザ）とウェブサーバ（ウェブサイト）の通信を中継するサーバである。 ウェブブラウザがウェブサイトにアクセスする際、リクエストをプロキシサーバに送り、プロキシサーバがリクエストに基づきウェブサーバと通信を行う。 ウェブサーバからプロキシサーバに送られたデータを、プロキシサーバがウェブブラウザへ転送する。 この仕組みにはいくつかのメリットがある。 1．キャッシング：プロキシサーバは一度アクセスしたウェブサイトのデータをキャッシュとして保存する。

QUESTION 117#original_knowledge_0059

Web閲覧中にウイルス感染を告げ、すぐにソフトを入れるよう促す警告が表示された。最も適切な初動はどれか。

  1. 0

    表示されたソフトを直ちにインストールする。

  2. 1

    警告のリンクを友人全員へ転送して確認してもらう。

  3. 2

    表示内のリンクを押さず、ブラウザを閉じて信頼できる経路で確認する。

  4. 3

    表示された電話番号に連絡して遠隔操作を許可する。

正答と解説を確認

正答2. 表示内のリンクを押さず、ブラウザを閉じて信頼できる経路で確認する。

解説

偽警告から不正ソフトを入れさせる手口があるため、表示の指示には従わず別経路で確認する。 URLをクリックしただけで一方的にサービスへの入会などの契約の成立を宣言し、使用料金を請求する詐欺をワンクリック詐欺という。 URLのクリックだけで契約が成立することはないため、無視してもよい。 ただし、不安な場合は警察や消費者センターなどに相談をするとよい。 コンピュータ・ウィルスなどの悪意あるソフトウェアをまとめてマルウェアという。 マルウェアはシステムを破壊するものだけではなく、ユーザに気づかれることなく情報を盗み取ったり、遠隔操作を行うものもある。 例えば、マルウェアの一種であるスパイウェアはキーボードからの入力やウェブブラウザを監視し、ユーザIDやパスワード、クレジットカード番号、閲覧履歴などを盗み出し、外部に送信する。 このような警告は、ユーザを驚かせることで、悪意あるウェブサイトやマルウェア（悪意あるソフトウェア）のインストールへと誘導している可能性が非常に高い。 そのため、警告メッセージが表示されても、無視するとよい。 これらのメッセージは広告のポップアップ機能を利用している可能性が高い。

QUESTION 118#original_knowledge_0060

ロボット型検索エンジンが検索結果を返す仕組みとして最も適切なものはどれか。

  1. 0

    検索のたびに全Webページへ直接問い合わせる。

  2. 1

    全てのWebページが必ず検索対象になる。

  3. 2

    人手だけで全サイトを分類して結果を返す。

  4. 3

    クローラで作った索引を、検索時に利用する仕組み。

正答と解説を確認

正答3. クローラで作った索引を、検索時に利用する仕組み。

解説

検索のたびに全Webページを巡回するのではなく、あらかじめ構築したインデックスを利用する。 ウェブ検索エンジンは、その場で世界中のウェブサイトから情報を収集するのではない。 ウェブクローラ（＝ウェブスパイダー、ロボット）と呼ばれるプログラムを使用し、定期的にインターネット上のウェブサイトを巡回し、データを収集する。 収集する情報はページの内容だけでなく、HTMLの構造を理解しページのタイトルや説明、キーワード、著者、作成日、最終更新日などの情報も読み取る。 ディレクトリ型の検索エンジンとは、人間がウェブページをカテゴリ分けしてデータベースに登録する方式であり、ウェブサイトの目録（リンク集）である。 手作業で一覧の作成を行うため、掲載されるウェブサイトが一定の品質以上のものであり、特定の分野や地域など目的のサイトがわかりやすく分類されているなどの利点がある。 過去、日本では、Yahoo!カテゴリが有名であったが、2018年にサービスを終了している。 通常の検索エンジンによって収集・インデックス化されないウェブサイトを深層ウェブ（deep web)という。 GoogleやMicrosoft Bingなどの検索エンジンでは、ウェブクローラ（＝ウェブスパイダー、ロボット）と呼ばれるプログラムを用いて、定期的にインターネット上のウェブサイトを巡回し情報を収集している。 しかし、クローラによる情報の取得を拒否しているサイトや会員専用ページや個人のストレージ、有料コンテンツ、パスワードで保護されているサイト、動的に生成されるサイト（Googleの検索結果など）などを巡回することはできず、ディープウェブとなる。

QUESTION 119#original_knowledge_0075

IoTとして最も適切な説明はどれか。

  1. 0

    ネットワークから全てのモノを切り離す考え方。

  2. 1

    記録をブロックにまとめて共有する台帳技術。

  3. 2

    モノをネットワークにつなぎ、データを活用する仕組み。

  4. 3

    Webサイトの掲載内容を保証する認証制度。

正答と解説を確認

正答2. モノをネットワークにつなぎ、データを活用する仕組み。

解説

IoTでは、モノがネットワークを通じてデータを交換し、遠隔操作や自動制御などに利用される。 IoT(Internet of Things)は、様々なデバイスやモノをインターネットに繋ぎ、情報の交換を可能にする仕組みである。 例えば、照明やエアコン、セキュリティカメラなどの家電製品がインターネットに接続され、遠隔操作ができるようになっている。 さらに、温度や湿度、明るさをセンサーで感知し、照明やエアコンが自動的に動作し、快適な状態を作り出すことができるスマートホームも作られている。 大きな視点では交通システムや電気供給、公共サービスなどをIoTにより効率化するスマートシティ構想がある。 身近な例では、スマートウォッチなどを使って心拍数などのバイタルデータを取得し、健康に関するアドバイスを受けられるウェアラブルデバイスもIoTのひとつである。

QUESTION 120#simple_original_140

インターネット上でIPパケットを届ける際、宛先となるネットワークインタフェースを論理的に識別するために用いるものはどれか。

  1. 0

    MACアドレス

  2. 1

    ポート番号

  3. 2

    IPアドレス

  4. 3

    URL

正答と解説を確認

正答2. IPアドレス

解説

IPアドレスは、IP通信の宛先となるネットワークインタフェースを論理的に識別する。MACアドレスやポート番号とは識別する対象が異なる。 MACアドレスは主に同一リンク内の配送、ポート番号は端末上のアプリケーション、URLはWeb資源の場所を識別する。IPアドレスはIPパケットの論理的な送信元・宛先を示す。

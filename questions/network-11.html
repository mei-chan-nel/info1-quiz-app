QUESTION 101#original_knowledge_0043

Wi-Fiとして最も適切な説明はどれか。

  1. 0

    有線LANだけで使う通信規格の総称。

  2. 1

    Bluetoothの正式名称。

  3. 2

    IEEE 802.11系の無線LAN機器に関する認証ブランド。

  4. 3

    Web通信を暗号化するプロトコル。

正答と解説を確認

正答2. IEEE 802.11系の無線LAN機器に関する認証ブランド。

解説

Wi‑Fiは、Wi‑Fi AllianceがIEEE 802.11系の無線LAN規格に基づく製品の相互接続性などを認証するブランドである。日常的には無線LANの呼称として広く使われるが、有線LANで用いられるEthernetや、近距離無線通信のBluetoothとは別の技術・用途である。

QUESTION 102#original_knowledge_0044

クライアントサーバ方式として最も適切な説明はどれか。

  1. 0

    全端末が常に同じ役割で資源を提供し合う方式。

  2. 1

    端末をネットワークから完全に切り離す方式。

  3. 2

    一台の端末だけで全利用者の画面を表示する方式。

  4. 3

    サービスを提供するサーバと、利用するクライアントに役割を分ける方式。

正答と解説を確認

正答3. サービスを提供するサーバと、利用するクライアントに役割を分ける方式。

解説

Webやメールなどでは、サーバが機能やデータを提供し、クライアントが利用する。 サービスを提供する側(サーバ)とサービスを要求する側(クライアント)を区別するシステムはクライアント・サーバシステムである。 クライアントサーバシステムはWebブラウジングや電子メールなどの多くのインターネットサービスで広く使われている。 分散処理とは、複数のコンピュータが協力してタスクを実行するシステムのことである。 クライアント・サーバシステムは分散処理を実現するためのひとつの方法ではあるが、分散処理では必ずしもクライアントとサーバを区別する必要はない。 クライアントとサーバの区別がない分散処理としては、複数のコンピュータが対等な関係で通信し合い、データや機能を共有するピアツーピアシステム(P2P)がある。

QUESTION 103#original_knowledge_0045

P2P方式として最も適切な説明はどれか。

  1. 0

    端末同士が対等な立場で機能やデータを提供し合う方式。

  2. 1

    全ての処理を中央サーバだけが行う方式。

  3. 2

    ネットワークを使わず端末内だけで完結する方式。

  4. 3

    端末のIPアドレスを自動的に割り当てる方式。

正答と解説を確認

正答0. 端末同士が対等な立場で機能やデータを提供し合う方式。

解説

P2Pでは、各端末がクライアントにもサーバにもなり得る。 データや機能を共有するために、複数のコンピュータが対等な関係で通信し合うシステムをピアツーピアシステム(Peer-to-Peer,P2P)という。 このとき、ネットワーク上の端末をノードという。 P2Pを利用したアプリケーションではノード間で直接通信を行うことで、中央のサーバを必要とせず、データの転送が高速化され、システム全体の負荷が分散されるため、効率的なデータ共有ができる。 P2Pのアプリケーションは多岐にわたり、ファイル共有（BitTorrentなど）やブロックチェーン技術（Bitcoinなどの仮想通貨）、VoIP（Voice over IP：LINE電話・Skypeなど）、ストリーミング配信などで使用されている。 P2Pでは匿名性を利用した違法コンテンツの配布、ノード内にデータが残り続けるため削除の困難さ、コンピュータウイルスを送り込まれるなどセキュリティ上の危険性、大量のデータがやり取りされることによる通信回線への負荷などの問題を孕んでいる。

QUESTION 104#original_knowledge_0046

クラウドコンピューティングとして最も適切な説明はどれか。

  1. 0

    利用者の端末内だけに資源を置く形態。

  2. 1

    ネットワーク経由で計算資源やソフトウェアを利用する形態。

  3. 2

    全端末を同じ部屋に置くための配線規格。

  4. 3

    通信を必ず暗号化する一つのプロトコル。

正答と解説を確認

正答1. ネットワーク経由で計算資源やソフトウェアを利用する形態。

解説

クラウドでは、利用者が必要に応じて遠隔の計算資源やサービスを利用できる。 クラウドコンピューティングは、ユーザが必要なときに必要な量の機能を提供するインターネット上のサービスである。 提供される機能には、ハードウェアリソース（CPU、ストレージ、メモリなど）やソフトウェアリソース（データベース、AI、アプリケーションなど）である。 ユーザは物理的なサーバを持つ必要や、自身のコンピュータにアプリケーションをインストールして管理する負担がなくなる。 これにより、需要に応じたスケールアップ・スケールダウンができ、最新の機能とセキュリティを使用することができる。 クラウドで何を提供するかで、SaaS、PaaS、IaaSに分類できる。

QUESTION 105#original_knowledge_0047

パケット交換方式として最も適切な説明はどれか。

  1. 0

    通信中は送受信者間の物理回線を占有し続ける方式。

  2. 1

    送信データを必ず一つの画像ファイルにする方式。

  3. 2

    データを小さなパケットに分け、ネットワークで順に転送する方式。

  4. 3

    全ての通信を一台の端末だけで処理する方式。

正答と解説を確認

正答2. データを小さなパケットに分け、ネットワークで順に転送する方式。

解説

インターネットはデータをパケットに分けて送るパケット交換を基本としている。 インターネットでは、通信のためにパケット交換方式を採用している。 パケット交換方式は、データをパケットという小さな部分に分割し、それらを独立して送信する方式である。 受信側では、送られてきたパケットを再度組み立て、データを復元する。 パケットに分割して送信することで、ネットワークの混雑状況に合わせて柔軟なルート変更や、別のコンピュータが送っているパケットと回線を共有しながら通信を行うことができる。 これにより、通信の信頼性と効率性を向上させている。

QUESTION 106#original_knowledge_0048

IPアドレスの主な役割として最も適切なものはどれか。

  1. 0

    同一機器内で通信先のアプリケーションを識別すること。

  2. 1

    利用者の氏名や所属などの個人情報を暗号化すること。

  3. 2

    Webページ内の見出しの階層構造を表すこと。

  4. 3

    ネットワーク上の通信先となる機器や接続点を識別すること。

正答と解説を確認

正答3. ネットワーク上の通信先となる機器や接続点を識別すること。

解説

IPアドレスは通信相手を特定するための論理アドレスである。 IPアドレスはインターネット上のデバイスに割り当てられた番号（住所）であり、通信を行う際の宛先として使用される。 より正確には、TCP/IPで通信を行うデバイスに割り当てられた番号である。 現在広く使われているIPアドレスはIPv4（バージョン4）である。 IPv4は32ビット（2進数32桁）の数である。 人間が見てわかりやすくするために、8ビットずつを「.」で区切り、0から255の10進数で表現される。

QUESTION 107#original_knowledge_0049

IPv6の説明として最も適切なものはどれか。

  1. 0

    128ビットで、IPv4より広いアドレス空間を持つ方式。

  2. 1

    16ビットのアドレスだけを用いる方式。

  3. 2

    IPv4より小さなアドレス空間を持つ方式。

  4. 3

    無線LAN専用の通信方式。

正答と解説を確認

正答0. 128ビットで、IPv4より広いアドレス空間を持つ方式。

解説

IPv6は128ビットのIPアドレスを用い、32ビットのIPv4よりはるかに広いアドレス空間を持つ。IPv4アドレスの不足へ対応することが導入目的の一つである。無線LAN専用ではなく、有線・無線を問わずIPネットワークで利用できる。

QUESTION 108#original_knowledge_0050

DNSの主な役割として最も適切なものはどれか。

  1. 0

    Webページを画像へ変換して配信すること。

  2. 1

    ドメイン名などの名前とIPアドレスを対応付けて調べること。

  3. 2

    電子メールの本文を暗号化すること。

  4. 3

    利用者の端末にソフトウェアを配布すること。

正答と解説を確認

正答1. ドメイン名などの名前とIPアドレスを対応付けて調べること。

解説

DNSは人が扱いやすい名前を、通信に使うIPアドレスへ対応付ける名前解決に用いる。 IPアドレス（例：142.250.196.99）は数字とドットの組み合わせであり、人間にとって覚えることは難しい。 そこで、人間にとって覚えやすく、扱いやすい識別子（名前）としてドメイン名（例：www.google.com）が考案された。 URLならhttps://www.example.co.jp/jouhou/index.htmlのexample.co.jpがドメイン名である。 ホスト名であるwwwも含め、www.example.co.jpを完全修飾ドメイン名という。 完全修飾ドメイン名を単にドメイン名ということもある。

QUESTION 109#original_knowledge_0051

TCPの特徴として最も適切なものはどれか。

  1. 0

    到達確認や順序制御を全く行わない。

  2. 1

    通信先のドメイン名をIPアドレスに変換する。

  3. 2

    確認応答や再送、順序制御によって信頼性を高める。

  4. 3

    Webページの内容を必ず暗号化する。

正答と解説を確認

正答2. 確認応答や再送、順序制御によって信頼性を高める。

解説

TCP（Transmission Control Protocol）はトランスポート層のプロトコルであり、確認応答、再送制御、順序制御、フロー制御などによって信頼性のあるバイトストリームをアプリケーションへ提供する。UDPは確認応答や再送、順序制御を標準では行わず、TCPより簡素なデータグラム通信を提供する。

QUESTION 110#original_knowledge_0052

ポート番号の主な役割として最も適切なものはどれか。

  1. 0

    世界中の通信先となる機器や接続点を識別する番号。

  2. 1

    利用者のパスワードを暗号化するために使う番号。

  3. 2

    文書を表示する際の文字コードを指定する番号。

  4. 3

    同じ機器上のアプリケーションやサービスを識別する番号。

正答と解説を確認

正答3. 同じ機器上のアプリケーションやサービスを識別する番号。

解説

Webやメールなど複数のサービスを一つの機器で動かすため、IPアドレスに加えてポート番号を使う。 1台のコンピュータには様々なアプリケーションが同時に動作している。 例えば、WebブラウザでWebサイトを見ながら、メールソフトでメールの送受信をするなどである。 ウェブサーバやメールサーバからクライアントのコンピュータまではIPアドレスをもとにデータが送られてくる。 サーバから送られてきたデータをクライアントがどのアプリケーションで受け取るかを指定するのがポート番号である。 つまり、ポート番号は通信が行われるアプリケーションを特定するための番号である。

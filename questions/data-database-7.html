QUESTION 061#center_style_curated_079_004

“注文”表の顧客番号は，“顧客”表の顧客番号を外部キーとして参照している。“注文”表に顧客A001の注文があるとき，参照の整合性を損なう操作として最も適当なものはどれか。

  1. 0

    “顧客”表から顧客A001の行を削除する。

  2. 1

    “注文”表に，すでに“顧客”表にある顧客の注文を追加する。

  3. 2

    “注文”表から，顧客A001の注文の行を削除する。

  4. 3

    “顧客”表に，新しい顧客C003の行を追加する。

正答と解説を確認

正答0. “顧客”表から顧客A001の行を削除する。

解説

外部キーは，別の表の主キーなどを参照して表どうしを関連付ける項目である。注文から参照されている顧客A001を顧客表から削除すると，注文表の顧客番号が存在しない顧客を指すことになる。

QUESTION 062#center_style_curated_079_005

“注文”表の顧客番号は，“顧客”表の顧客番号を外部キーとして参照している。“顧客”表にA001とB002だけが登録されているとき，参照の整合性を損なう操作として最も適当なものはどれか。

  1. 0

    “注文”表に，顧客番号C003の注文を追加する。

  2. 1

    “注文”表に，顧客番号A001の注文を追加する。

  3. 2

    “顧客”表に，顧客番号C003の行を追加する。

  4. 3

    “注文”表から，顧客番号B002の注文を削除する。

正答と解説を確認

正答0. “注文”表に，顧客番号C003の注文を追加する。

解説

外部キーには，参照先の表に存在する値を入れる必要がある。顧客表にA001とB002だけがあるなら，注文表に顧客番号C003を追加すると存在しない顧客を参照することになり，参照の整合性を損なう。

QUESTION 063#center_style_curated_080_002

表Rから列A1，A3，A5だけを取り出すSQL文「SELECT A1, A3, A5 FROM R」に対応する関係データベースの操作として，最も適当なものはどれか。

  1. 0

    結合

  2. 1

    差

  3. 2

    射影

  4. 3

    直積

正答と解説を確認

正答2. 射影

解説

射影は，表から特定の列だけを取り出す操作である。SELECT A1, A3, A5 FROM R はRの行全体ではなく列A1，A3，A5を指定しているので射影に当たる。

QUESTION 064#center_style_curated_080_004

関係データベースの主キーに必要な条件として，値が重複しないことのほかに最も適当なものはどれか。

  1. 0

    値が空でないこと

  2. 1

    必ず1つの列だけで作ること

  3. 2

    表の先頭の列であること

  4. 3

    別の表の主キーと必ず同じ値であること

正答と解説を確認

正答0. 値が空でないこと

解説

主キーは表の各行を一意に識別するための項目なので，値が重複せず，空でないことが必要である。

QUESTION 065#center_style_curated_080_005

利用者情報を検索して表示するだけのアプリに，データベースへのアクセス権を与える。セキュリティ上，最も適当な権限はどれか。

  1. 0

    管理者権限

  2. 1

    更新権限

  3. 2

    参照権限

  4. 3

    参照権限と更新権限の両方

正答と解説を確認

正答2. 参照権限

解説

検索して表示するだけなら，必要なのはデータを読むための参照権限である。必要以上の権限を与えないことは，誤操作や不正な変更を防ぐ上で重要である。

QUESTION 066#center_style_curated_082_006

  1. 0

    同じ内容のデータの重複を減らし，更新時の不整合を起こりにくくする。

QUESTION 068#center_style_curated_083_009

リレーショナルデータベースで，表の行を一意に識別するための項目と，表どうしを関連付けるときに使う項目の組合せとして，最も適当なものはどれか。

  1. 0

    主キー，外部キー

  2. 1

    レコード，文字列

  3. 2

    フィールド，主キー

  4. 3

    ディレクトリ，ファイル

正答と解説を確認

正答0. 主キー，外部キー

解説

主キーは，表の各行を一意に識別する列または列の組である。外部キーは，他の表の主キーなどを参照する列または列の組であり，表どうしを関連付けるために使う。したがって，組合せは「主キー，外部キー」である。

QUESTION 069#center_style_curated_083_017

部署表（部署コード，部署名）と社員表（社員コード，社員名，部署コード）があり，社員は必ず部署表に存在する部署に所属する。主キーと外部キーの設定として，最も適当なものはどれか。

  1. 0

    部署表の部署コードと社員表の社員コードを主キーにし，社員表の部署コードを外部キーにする。

  2. 1

    部署表の部署名と社員表の社員名を主キーにし，社員表の社員コードを外部キーにする。

  3. 2

    社員表の部署コードだけを主キーにし，外部キーは設定しない。

  4. 3

    部署表にも社員表にも主キーを設定せず，部署名だけで関連付ける。

正答と解説を確認

正答0. 部署表の部署コードと社員表の社員コードを主キーにし，社員表の部署コードを外部キーにする。

解説

部署表では部署コードで部署を一意に識別し，社員表では社員コードで社員を一意に識別する。社員表の部署コードを部署表の部署コードに対応する外部キーにすると，存在しない部署への所属を防ぎやすい。

QUESTION 091#simple_original_167

名義尺度で表したデータとして最も適切なものはどれか。

  1. 0

    利用者の満足度を低い・普通・高いの順に並べたデータ。

  2. 1

    利用者を居住地域A・B・Cのように、順序を持たない分類名で区分したデータ。

  3. 2

    気温を摂氏で測定したデータ。

  4. 3

    荷物の質量をkgで測定したデータ。

正答と解説を確認

正答1. 利用者を居住地域A・B・Cのように、順序を持たない分類名で区分したデータ。

解説

名義尺度は、対象を種類や名前で分類する尺度であり、値の大小や間隔に意味はない。居住地域の区分には、地域間の順序や差を定めない。 居住地域A・B・Cは名前による分類であり、地域間に大小や等しい間隔を定めない。このようなデータが名義尺度であり、順序・間隔・比例の尺度とは異なる。

QUESTION 092#simple_original_168

順序尺度で表したデータとして最も適切なものはどれか。

  1. 0

    利用者を所属する部活動名で分類したデータ。

  2. 1

    気温を摂氏で測定したデータ。

  3. 2

    商品の評価を不満・やや不満・普通・満足・とても満足の順に区分したデータ。

  4. 3

    走行距離をkmで測定したデータ。

正答と解説を確認

正答2. 商品の評価を不満・やや不満・普通・満足・とても満足の順に区分したデータ。

解説

順序尺度は、順序には意味があるが、隣り合う区分の差が等しいとは限らない尺度である。満足ととても満足の差を、普通と満足の差と同じ大きさとはいえない。 満足度には低いから高いという順序があるが、隣り合う区分の差が等しいとは限らない。このため順序尺度であり、量の比を扱う比例尺度ではない。

QUESTION 093#simple_original_169

間隔尺度で表したデータとして最も適切なものはどれか。

  1. 0

    利用者を血液型で分類したデータ。

  2. 1

    大会の順位を1位・2位・3位で示したデータ。

  3. 2

    水の量をLで測定したデータ。

  4. 3

    教室の気温を摂氏で測定したデータ。

正答と解説を確認

正答3. 教室の気温を摂氏で測定したデータ。

解説

間隔尺度は差に意味があるが、原点が任意であるため比には意味がない。摂氏20度は10度より10度高いが、2倍の温度とはいえない。 摂氏温度は差に意味があるため、20度は10度より10度高いといえる。一方で0度は温度の不存在を示さないため、20度を10度の2倍とはいえない。

QUESTION 094#simple_original_170

比例尺度で表したデータとして最も適切なものはどれか。

  1. 0

    荷物の質量をkgで測定したデータ。

  2. 1

    利用者を居住地域名で分類したデータ。

  3. 2

    アンケートの順位を1位・2位・3位で示したデータ。

  4. 3

    室温を摂氏で測定したデータ。

正答と解説を確認

正答0. 荷物の質量をkgで測定したデータ。

解説

比例尺度は絶対的な0を持ち、差だけでなく比にも意味がある。例えば4kgは2kgの2倍の質量であり、0kgは質量がないことを表す。 質量は0kgを基準にでき、4kgは2kgの2倍と解釈できる。このように絶対的なゼロと比に意味を持つデータが比例尺度である。

QUESTION 095#simple2_original_063

Web APIから次のデータを受け取った。このように、キーと値の組で表されたテキスト形式はどれか、次から選べ。 {"city":"Tokyo","temperature":28}

  1. 0

    HTML

  2. 1

    CSS

  3. 2

    CSV

  4. 3

    JSON

正答と解説を確認

正答3. JSON

解説

提示されたデータはJSON（JavaScript Object Notation）形式である。波括弧で囲まれた部分はオブジェクトであり、cityやtemperatureがキー、Tokyoや28がそれぞれの値である。複数の組はカンマで区切る。 JSONは文字列、数値、真偽値、配列、オブジェクトなどを組み合わせて階層的なデータを表せる。人にも比較的読みやすく、多くのプログラミング言語で扱えるため、Web APIのデータ交換によく用いられる。

QUESTION 096#simple2_original_064

ソフトから出力したデータが次のようにカンマで区切られた形式であった。このデータ形式を何というか、次から選べ。 学籍番号,氏名,得点 101,佐藤,82

  1. 0

    CSV

  2. 1

    JSON

  3. 2

    XML

  4. 3

    ZIP

正答と解説を確認

正答0. CSV

解説

CSVはComma-Separated Valuesの略で、表形式のデータをテキストとして保存する形式である。通常、一行が一件のレコード、カンマで区切られた各値が列の項目に対応する。提示例では、1行目が項目名、2行目が一人分のデータである。 構造が単純で多くの表計算ソフトやデータ処理ソフトから入出力できる一方、文字コード、改行、値にカンマや改行を含む場合の引用符の扱いなどを、やり取りする側でそろえる必要がある。

QUESTION 097#simple2_original_104

情報システムやデータベースに関する説明として、誤っているものを選べ。

  1. 0

    データベースは、複数のデータを一定の形式で整理して蓄積し、検索や更新をしやすくしたものである。

  2. 1

    POSシステムでは、販売した商品や数量、時刻などのデータを記録し、売上分析や在庫管理に活用する。

  3. 2

    銀行の振込処理で、送金元口座からの出金だけが完了し、送金先口座への入金に失敗した場合は、出金だけを確定する。

  4. 3

    情報システムに蓄積されたデータを分析し、業務の改善や新しいサービスの提供に活用することがある。

正答と解説を確認

正答2. 銀行の振込処理で、送金元口座からの出金だけが完了し、送金先口座への入金に失敗した場合は、出金だけを確定する。

解説

誤っているのは、送金先への入金に失敗しても、送金元からの出金だけを確定するという説明である。振込では、出金と入金を分割できない一連の処理として扱い、すべて成功したときだけ確定する。一部が失敗した場合は、それまでの変更を取り消して処理前の状態へ戻す。 この「すべて実行するか、まったく実行しないか」という性質をトランザクションの原子性という。関連する更新を一括管理することで、残高などのデータの整合性を保つ。

QUESTION 098#simple2_original_106

情報システムとデータベースに関する説明として、誤っているものを選べ。

  1. 0

    POSシステムは、レジにある現金の残高を管理することを主な目的とし、販売した商品や数量の情報は記録しない。

  2. 1

    銀行の振込処理では、出金と入金を一まとまりの処理として扱い、口座残高に不整合が生じないようにする。

  3. 2

    データベースは、データを一定の形式で整理して蓄積し、検索や更新をしやすくするために用いられる。

  4. 3

    店舗に蓄積された販売データを分析すると、商品の売れ方や時間帯ごとの売上傾向を把握できる。

正答と解説を確認

正答0. POSシステムは、レジにある現金の残高を管理することを主な目的とし、販売した商品や数量の情報は記録しない。

解説

POSシステムは、商品を販売した時点で商品コード、数量、価格、時刻などを記録する仕組みであり、現金残高だけを管理するものではない。したがって、選択肢0が誤りである。 販売データをデータベースに蓄積すると、売上集計、在庫管理、仕入れ計画、時間帯別の需要分析などに活用できる。銀行の振込のように複数の更新を一体として扱う処理では、途中だけが実行されて不整合が生じないよう、トランザクションとして管理する。

QUESTION 099#simple2_original_107

複数の利用者が同じデータを更新しても、一貫した状態を保つためのデータベース機能に当たるものを、次から選べ。

  1. 0

    障害でデータが壊れても、バックアップから復旧できる。

  2. 1

    同じデータを同時に更新できないよう、処理中の行をロックする。

  3. 2

    SQLというデータベース言語でデータを操作する。

  4. 3

    利用者ごとにアクセスできる範囲を設定する。

正答と解説を確認

正答1. 同じデータを同時に更新できないよう、処理中の行をロックする。

解説

複数の処理が同じデータを同時に更新すると、後の更新が前の更新を上書きするなどして、データに矛盾が生じることがある。処理中の行をロックし、他の処理による同時更新を制限する排他制御は、このような競合を防いで一貫性を保つための機能である。

QUESTION 100#simple2_original_108

生徒表と試験結果表を生徒IDで関連付けて管理する。リレーショナルデータベースの特徴として最も適切なものを、次から選べ。

  1. 0

    すべてのデータを一つの文章として保存する。

  2. 1

    データを親子関係だけのツリー構造で管理する。

  3. 2

    表の間に関連を持たせず、同じ生徒情報を各表へ重複して登録し、表ごとに更新する。

  4. 3

    データを表形式で管理し、共通する項目を使って複数の表を関連付けられる。

正答と解説を確認

正答3. データを表形式で管理し、共通する項目を使って複数の表を関連付けられる。

解説

リレーショナルデータベースは、データを行と列からなる表で管理し、主キーと外部キーなどの共通項目を使って表同士を関連付ける。生徒表と試験結果表を生徒IDで結び付ければ、生徒の基本情報を何度も重複して保存せずに、各生徒の試験結果を取り出せる。

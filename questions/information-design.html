QUESTION 001#center_style_common_test_0019

マウスカーソルを対象物まで移動する時間について，次の性質がある。 ・対象物が大きいほど，移動時間は短い。 ・対象物までの距離が短いほど，移動時間は短い。 ・画面端に接する対象物は，カーソルが画面外に出ないため実質的に選択しやすい。 最も短時間で選択しやすい対象物を，次の選択肢から一つ選べ。

  1. 0

    カーソルから遠く，画面中央にある小さな対象物

  2. 1

    カーソルから遠いが，画面中央にある大きな対象物

  3. 2

    カーソルに近く，画面端に接している対象物

  4. 3

    カーソルに近いが，非常に小さく画面中央にある対象物

正答と解説を確認

正答2. カーソルに近く，画面端に接している対象物

解説

示された性質では，対象物が近いほど，また画面端に接しているほど選択しやすい。正答の内容は距離が短く，画面端にも接しているため，最も短時間で選択しやすい。

QUESTION 002#center_style_common_test_0020

すべて同じ大きさのメニュー項目を，利用頻度に基づいて配置する。利用頻度の低い項目の配置として最も適当なものを，次の選択肢から一つ選べ。

  1. 0

    カーソルに最も近い場所に配置する。

  2. 1

    カーソルから遠い場所に配置する。

  3. 2

    画面端に接する場所に配置する。

  4. 3

    最も大きく目立つように表示する。

正答と解説を確認

正答1. カーソルから遠い場所に配置する。

解説

同じ大きさなら，カーソルに近い場所や画面端は選択しやすい。利用頻度の低い項目は，頻繁に使う項目より選びやすい場所を優先する必要が低いため，カーソルから遠い場所に置くのが適当である。

QUESTION 003#center_style_common_test_0031

1月から12月までを円形に並べた入力ホイールがあり，12月の次は1月，1月の前は12月となる。各月が選ばれる確率がすべて等しいとする。 初期値と，選択に必要な平均移動距離の関係として最も適当なものを，次の選択肢から一つ選べ。

  1. 0

    1月を初期値にすると最小になる。

  2. 1

    12月を初期値にすると最小になる。

  3. 2

    1月または7月を初期値にすると最小になる。

  4. 3

    どの月を初期値にしても平均移動距離は変わらない。

正答と解説を確認

正答3. どの月を初期値にしても平均移動距離は変わらない。

解説

1月から12月が円形に並び，各月が同じ確率で選ばれるなら，どの月を初期値にしても周囲にある月の距離の並びは回転しただけで同じになる。平均移動距離も同じである。

QUESTION 004#center_style_common_test_0043

文書中で，見出しには「*」，小見出しには「**」，箇条書きには「-」，重要性の低い補足情報には「#」を行頭に付けている。 文書の構造を分かりやすくする装飾として最も適当なものを，次の選択肢から一つ選べ。

  1. 0

    見出しを小さくし，箇条書きを太字にする。

  2. 1

    見出しを大きく太字にし，補足情報を小さく細い文字にする。

  3. 2

    補足情報を最も大きくし，見出しを目立たなくする。

  4. 3

    すべての行を同じ大きさ，同じ太さで表示する。

正答と解説を確認

正答1. 見出しを大きく太字にし，補足情報を小さく細い文字にする。

解説

見出しは文書構造の上位にあるため目立たせ，重要性の低い補足情報は本文より控えめに表示すると，構造を読み取りやすくなる。

QUESTION 005#center_style_common_test_0044

見出し，小見出し，本文からなる文書の階層構造を分かりやすくする方法として最も適当なものを，次の選択肢から一つ選べ。

  1. 0

    多種類のフォントと文字色を使用する。

  2. 1

    行間をできるだけ狭くする。

  3. 2

    小見出しとその内容にインデントを設定する。

  4. 3

    人名などの固有名詞を伏せ字にする。

正答と解説を確認

正答2. 小見出しとその内容にインデントを設定する。

解説

インデントを使うと，小見出しとその内容が上位の見出しに属していることを視覚的に示せるため，階層構造が分かりやすくなる。

QUESTION 006#center_style_ip_curated_007

アクセシビリティに配慮したソフトウェア設計の説明として最も適当なものはどれか。

  1. 0

    高齢者や障害のある人を含め，様々な人が利用できるようにする。

  2. 1

    画面の文字やボタンを小さくして，一度に多くの情報を表示する。

  3. 2

    利用者の許可なく，個人情報を公開する。

  4. 3

    色だけで状態を区別し，文字では説明しない。

正答と解説を確認

正答0. 高齢者や障害のある人を含め，様々な人が利用できるようにする。

解説

アクセシビリティは，年齢や障害の有無などにかかわらず，できるだけ多くの人が利用しやすいようにする考え方である。

QUESTION 007#center_style_ip_curated_013

丸いドアノブを見ると回して開けるものだと分かるように，ものの形や見た目から利用者が可能な操作を想起できる性質を，UIデザインでは何と呼ぶか。

  1. 0

    アフォーダンス

  2. 1

    個人情報

  3. 2

    著作権

  4. 3

    暗号化

正答と解説を確認

正答0. アフォーダンス

解説

アフォーダンスは，ものの形や見た目などから，利用者がどのような操作を行えるかを想起できる性質として説明される。丸いドアノブから「回せる」と分かる例がこれに当たる。

QUESTION 009#center_style_ap_curated_017

Webページのアクセシビリティを高める設計として最も適当なものはどれか。

  1. 0

    ページ表示時に音声を必ず自動再生する

  2. 1

    見た目を整える目的だけで表組みを使う

  3. 2

    入力必須項目は色だけでなく、項目名の近くに「必須」と文字でも示す

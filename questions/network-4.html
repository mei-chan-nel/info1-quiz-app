QUESTION 098#original_knowledge_0040

LANとして最も適切な説明はどれか。

  1. 0

    地理的に離れたLAN同士を結ぶ広域ネットワーク。

  2. 1

    Web上の名前をIPアドレスに対応付ける仕組み。

  3. 2

    Web通信を暗号化するためのプロトコル。

  4. 3

    学校や家庭など、比較的限られた範囲を結ぶネットワーク。

正答と解説を確認

正答3. 学校や家庭など、比較的限られた範囲を結ぶネットワーク。

解説

LANはLocal Area Networkの略で、限られた場所にある機器を接続するネットワークである。 LAN(Local Area Network)は家庭内や建物などの一定の狭い範囲内における、コンピュータやサーバ、プリンター、ゲーム機などの機器が接続されたネットワークである。 LANの構築・管理はユーザ自らが行う。 離れた場所にある複数のLAN同士を接続するネットワークをWAN(Wide Area Network：広域通信網)という。 さらに、不特定多数のネットワーク同士を繋いだ巨大なネットワークがインターネットである。 インターネットへの接続するにはISP（Internet Service Provider）と契約する必要がある。

QUESTION 102#original_knowledge_0044

クライアントサーバ方式として最も適切な説明はどれか。

  1. 0

    全端末が常に同じ役割で資源を提供し合う方式。

  2. 1

    端末をネットワークから完全に切り離す方式。

  3. 2

    一台の端末だけで全利用者の画面を表示する方式。

  4. 3

    サービスを提供するサーバと、利用するクライアントに役割を分ける方式。

正答と解説を確認

正答3. サービスを提供するサーバと、利用するクライアントに役割を分ける方式。

解説

Webやメールなどでは、サーバが機能やデータを提供し、クライアントが利用する。 サービスを提供する側(サーバ)とサービスを要求する側(クライアント)を区別するシステムはクライアント・サーバシステムである。 クライアントサーバシステムはWebブラウジングや電子メールなどの多くのインターネットサービスで広く使われている。 分散処理とは、複数のコンピュータが協力してタスクを実行するシステムのことである。 クライアント・サーバシステムは分散処理を実現するためのひとつの方法ではあるが、分散処理では必ずしもクライアントとサーバを区別する必要はない。 クライアントとサーバの区別がない分散処理としては、複数のコンピュータが対等な関係で通信し合い、データや機能を共有するピアツーピアシステム(P2P)がある。

QUESTION 103#original_knowledge_0045

P2P方式として最も適切な説明はどれか。

  1. 0

    端末同士が対等な立場で機能やデータを提供し合う方式。

  2. 1

    全ての処理を中央サーバだけが行う方式。

  3. 2

    ネットワークを使わず端末内だけで完結する方式。

  4. 3

    端末のIPアドレスを自動的に割り当てる方式。

正答と解説を確認

正答0. 端末同士が対等な立場で機能やデータを提供し合う方式。

解説

P2Pでは、各端末がクライアントにもサーバにもなり得る。 データや機能を共有するために、複数のコンピュータが対等な関係で通信し合うシステムをピアツーピアシステム(Peer-to-Peer,P2P)という。 このとき、ネットワーク上の端末をノードという。 P2Pを利用したアプリケーションではノード間で直接通信を行うことで、中央のサーバを必要とせず、データの転送が高速化され、システム全体の負荷が分散されるため、効率的なデータ共有ができる。 P2Pのアプリケーションは多岐にわたり、ファイル共有（BitTorrentなど）やブロックチェーン技術（Bitcoinなどの仮想通貨）、VoIP（Voice over IP：LINE電話・Skypeなど）、ストリーミング配信などで使用されている。 P2Pでは匿名性を利用した違法コンテンツの配布、ノード内にデータが残り続けるため削除の困難さ、コンピュータウイルスを送り込まれるなどセキュリティ上の危険性、大量のデータがやり取りされることによる通信回線への負荷などの問題を孕んでいる。

QUESTION 105#original_knowledge_0047

QUESTION 011#center_style_common_test_0036

画像の各画素をビット列で表し，残したい部分をすべて1，消したい部分をすべて0としたマスク画像を用いる。 元の画像から，マスク画像で1になっている部分だけを取り出すための演算として最も適当なものを，次の選択肢から一つ選べ。

  1. 0

    AND演算

  2. 1

    OR演算

  3. 2

    AND演算の後にNOT演算

  4. 3

    OR演算の後にNOT演算

正答と解説を確認

正答0. AND演算

解説

AND演算では，マスクが1の部分は元の値がそのまま残り，マスクが0の部分は0になる。そのため，1になっている部分だけを取り出すにはAND演算を使う。

QUESTION 012#center_style_common_test_0037

グレースケール画像のヒストグラムには，被写体に対応する低～中階調側の山と，背景に対応する高階調側の山があり，二つの山の間には画素数の少ない谷がある。 背景だけを選択するために指定する範囲として最も適当なものを，次の選択肢から一つ選べ。

  1. 0

    最も暗い階調から，被写体の山の終わりまで

  2. 1

    被写体の山の範囲だけ

  3. 2

    二つの山の間の谷より高階調側の範囲

  4. 3

    最も暗い階調から最も明るい階調までの全範囲

正答と解説を確認

正答2. 二つの山の間の谷より高階調側の範囲

解説

背景に対応する山が高階調側にあるなら，二つの山の間の谷を境にして，それより高い階調側を選ぶと背景を取り出しやすい。

QUESTION 013#center_style_common_test_0038

背景画像を (B)，合成する被写体画像を (C)，被写体部分が1，それ以外が0であるマスク画像を (M) とする。 背景画像に被写体を合成する論理式として最も適当なものを，次の選択肢から一つ選べ。

  1. 0

    ((B AND M) OR (C AND M))

  2. 1

    ((B AND NOT M) OR (C AND M))

  3. 2

    ((B OR NOT M) AND (C OR M))

  4. 3

    ((B OR M) AND (C AND M))

正答と解説を確認

正答1. ((B AND NOT M) OR (C AND M))

解説

マスクMが1の被写体部分ではCを使い，Mが0のそれ以外の部分ではBを使えばよい。式 ((B AND NOT M) OR (C AND M)) は，M=0の場所ではB AND 1で背景Bが残り，M=1の場所ではC AND 1で被写体Cが残る。

QUESTION 014#center_style_common_test_0039

24時間制の時刻について，時と分を別々のビット列で表す。時を表すビット数と分を表すビット数をそれぞれ最小にするとき，その合計はいくつか。

  1. 0

    9ビット

  2. 1

    10ビット

  3. 2

    11ビット

  4. 3

    12ビット

  5. 4

    13ビット

正答と解説を確認

正答2. 11ビット

解説

24時間制の時は0～23の24通りなので，2⁴=16では不足し，2⁵=32で足りるため5ビット必要である。分は0～59の60通りなので，2⁵=32では不足し，2⁶=64で足りるため6ビット必要である。合計は5+6=11ビットである。

QUESTION 015#center_style_common_test_0053

24fpsで作られた動画のすべてのフレームを，フレームを追加したり削除したりせずに60fpsで再生する。 元の動画と比べた再生時間として最も適当なものを，次の選択肢から一つ選べ。

  1. 0

    変わらない。

  2. 1

    2.5倍になる。

  3. 2

    0.4倍になる。

  4. 3

    1/24倍になる。

正答と解説を確認

正答2. 0.4倍になる。

解説

フレーム数を変えない場合，再生時間は「フレーム数÷fps」で決まる。同じフレーム数を24fpsではなく60fpsで再生するので，再生時間の比は24÷60=0.4である。

QUESTION 016#center_style_common_test_0054

解像度1920×1080，24ビットフルカラー，24fps，2時間の動画を，圧縮せずに保存する。1TB＝10¹²バイトとして，データ量に最も近いものを，次の選択肢から一つ選べ。

  1. 0

    約1GB

  2. 1

    約10GB

  3. 2

    約100GB

  4. 3

    約1TB

正答と解説を確認

正答3. 約1TB

解説

1フレームの画素数は1920×1080=2,073,600画素である。24ビットフルカラーは1画素3バイトなので，1フレームは約6,220,800バイトになる。2時間は7200秒で，24fpsでは24×7200=172,800フレームである。全体は約6,220,800×172,800=約1.07×10¹²バイト，つまり約1TBである。

QUESTION 017#center_style_common_test_0055

60fpsの動画が360フレームで構成されている。この動画の再生時間を，次の選択肢から一つ選べ。

  1. 0

    4秒

  2. 1

    6秒

  3. 2

    8秒

  4. 3

    10秒

正答と解説を確認

正答1. 6秒

解説

60fpsは1秒間に60フレームを再生するという意味である。360フレームの再生時間は360÷60=6秒である。

QUESTION 018#center_style_common_test_0056

列車が通過する様子を撮影する。次のうち，フレーム間の変化する部分だけを記録する圧縮方式で，データ量が最も小さくなりやすい撮影方法を選べ。

  1. 0

    カメラを三脚に固定し，列車が通過する様子を撮影する。

  2. 1

    列車が画面中央に来るよう，常にカメラを動かして撮影する。

  3. 2

    撮影中に繰り返し拡大と縮小を行う。

  4. 3

    撮影中にカメラを不規則に上下左右へ動かす。

正答0. カメラを三脚に固定し，列車が通過する様子を撮影する。

フレーム間で変化した部分だけを記録する圧縮では，背景がほとんど変わらず，被写体だけが変化する動画ほどデータ量を小さくしやすい。カメラを三脚に固定すれば，背景の位置が安定する。

QUESTION 019#center_style_common_test_0057

圧縮していない24ビットフルカラー，60fpsの動画に，次の三つの変換をそれぞれ行った。 ・あ：横と縦の画素数をそれぞれ半分にする。 ・い：再生時間を変えず，30fpsにする。 ・う：256階調のグレースケールにする。 変換後のデータ量の大小関係として最も適当なものを，次の選択肢から一つ選べ。

  1. 0

    あ＜い＜う

  2. 1

    あ＜う＜い

  3. 2

    い＜あ＜う

  4. 3

    う＜い＜あ

正答と解説を確認

正答1. あ＜う＜い

解説

基準のデータ量に対して，あは横と縦を半分にするので画素数が1/2×1/2=1/4になる。いはfpsを60から30にするのでフレーム数が1/2になる。うは24ビットフルカラーを256階調，つまり8ビットにするので1/3になる。大小は1/4＜1/3＜1/2なので，あ＜う＜いとなりこの内容が正答である。

QUESTION 020#center_style_common_test_0058

24fpsの動画とともに，サンプリング周波数48kHzの音声を記録する。 動画の1フレームを表示する時間の間に，音声は1チャンネル当たり何回標本化されるか。最も適当なものを，次の選択肢から一つ選べ。

  1. 0

    24回

  2. 1

    500回

  3. 2

    2000回

  4. 3

    48000回

正答と解説を確認

正答2. 2000回

解説

24fpsでは1フレームの時間は1/24秒である。48kHzは1秒間に48,000回標本化するという意味なので，1フレームの時間に行われる標本化回数は48,000÷24=2,000回である。

QUESTION 061#center_style_curated_067_015

URLのドメイン名ではWebページを開けないが，同じWebサーバのIPアドレスを直接指定すると開けた。この原因として最も考えられるものはどれか。

  1. 0

    DNSサーバが，ドメイン名をIPアドレスに対応付けられていない。

  2. 1

    PCの画面の解像度が低すぎる。

  3. 2

    キーボードの入力方式が日本語になっている。

  4. 3

    Webページの文字の大きさが小さい。

正答と解説を確認

正答0. DNSサーバが，ドメイン名をIPアドレスに対応付けられていない。

解説

ドメイン名で接続できず，IPアドレスを直接指定すると接続できるなら，ドメイン名をIPアドレスへ対応付けるDNSに問題がある可能性が高い。

QUESTION 065#center_style_curated_068_008

Webブラウザにドメイン名を含むURLを入力すると，対応するWebサーバのIPアドレスが調べられた。この名前解決を行う仕組みはどれか。

  1. 0

    DNS

  2. 1

    DHCP

  3. 2

    NAT

  4. 3

    NTP

正答と解説を確認

正答0. DNS

解説

DNSは，ドメイン名やホスト名とIPアドレスを対応付ける仕組みである。利用者がURLにドメイン名を入力すると，DNSによる名前解決で通信先のIPアドレスが調べられ，Webサーバへ接続できる。

QUESTION 067#center_style_curated_068_012

アクセス数が時期によって大きく変動するWebサービスで，最大時の負荷に合わせたサーバを年間を通して保有することを避けたい。クラウドコンピューティングの利用方法として，最も適当なものはどれか。

  1. 0

    需要に応じてサーバ資源を増減し，利用量に応じた構成で運用する。

  2. 1

    ネットワークから切り離し，アクセス要求を紙に印刷して処理する。

  3. 2

    最小時の負荷に合わせてサーバ資源を固定し，稼働状況は監視しない。

  4. 3

    管理を簡単にするため，全運用担当者が一つの管理者アカウントを共有する。

正答と解説を確認

正答0. 需要に応じてサーバ資源を増減し，利用量に応じた構成で運用する。

解説

クラウドコンピューティングでは，ネットワーク経由でコンピュータ資源を必要に応じて利用できる。需要に応じて資源を増減する伸縮性を活用すれば，最大負荷に合わせた設備を常時保有せずに，負荷の変動へ対応しやすくなる。

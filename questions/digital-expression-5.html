QUESTION 041#center_style_ip_curated_092

2桁の2進数A=11、B=11を加算する。結果を4の位、2の位、1の位の順に表すとどうなるか。

  1. 0

    010

  2. 1

    011

  3. 2

    101

  4. 3

    110

正答と解説を確認

正答3. 110

解説

2進数11は10進数で3である。3+3=6で、6を2進数で表すと110になる。

QUESTION 042#center_style_ap_curated_081

8ビットの2の補数表現で表せる整数の範囲として最も適当なものはどれか。

  1. 0

    -255から255

  2. 1

    -128から127

  3. 2

    -127から128

  4. 3

    0から255

正答と解説を確認

正答1. -128から127

解説

nビットの2の補数表現では、表せる整数の範囲は－2⁽ⁿ⁻¹⁾から2⁽ⁿ⁻¹⁾－1である。8ビットでは－128から127になる。符号なし整数の範囲と混同しないようにする。

  1. 0

    展開すると，圧縮前のデータと完全に同じデータに戻る圧縮である。

  2. 1

    人が気付きにくい情報を一部捨てるなどして，元と完全には同じに戻らない圧縮である。

QUESTION 045#center_style_fe_curated_070

8ビットの符号なし整数で表せる最大値は255である。この形式で 250+10 を計算しようとしたときに起こることとして最も適当なものはどれか。

  1. 0

    計算結果260が表せる範囲を超えるので，オーバーフローが起こる。

  2. 1

    8ビットでは必ず小数になるので，けた落ちが起こる。

  3. 2

    250と10はどちらも整数なので，必ず正確に260を保存できる。

  4. 3

    計算前の250が255以下なので，計算後も範囲を超えることはない。

正答と解説を確認

正答0. 計算結果260が表せる範囲を超えるので，オーバーフローが起こる。

解説

8ビットの符号なし整数で表せる範囲は0から255までである。250+10=260はこの範囲を超えるため，オーバーフローが起こる。けた落ちは近い値同士の引き算などで有効な桁が失われる現象であり，この場合とは違う。

QUESTION 047#center_style_ap_curated_085

ステレオ音声を標本化周波数12kHz、量子化ビット数12ビットで5秒間記録し、その後、データ量を3分の1に圧縮する。圧縮後のデータ量は何kバイトか。ただし、ステレオは2チャンネル、1kバイト＝1,000バイトとする。

  1. 0

    30kバイト

  2. 1

    60kバイト

  3. 2

    180kバイト

  4. 3

    360kバイト

正答と解説を確認

正答1. 60kバイト

解説

圧縮前は12,000回/秒×12ビット×5秒×2チャンネル＝1,440,000ビットで、180,000バイト＝180kバイトである。3分の1に圧縮すると60kバイトになる。
